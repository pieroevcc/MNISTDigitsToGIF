# v0 — Baseline Run (base 64, no EMA)

Reference snapshot of the first full training run, captured so we can measure whether
the planned improvements (EMA weights + `--base 96`) actually help. Run on 2026-06-17.

## Hardware
- GPU: NVIDIA GeForce RTX 4060 Laptop GPU (8 GB)
- PyTorch 2.12.1+cu126, Python 3.12.10, Windows 11

## Configuration (README defaults)
| Param | Value | Notes |
|-------|-------|-------|
| `--size` | 32 | frame H×W |
| `--frames` | 10 | clip length T |
| `--digits` | 1 | one bouncing digit per clip |
| `--base` | 64 | U-Net base channels |
| `--mults` | 1 2 4 | channel multipliers |
| `--timesteps` | 1000 | DDPM diffusion steps |
| `--batch` | 16 | fit ~4.1 GB / 8 GB VRAM |
| `--length` | 20000 | synthetic clips per epoch |
| `--epochs` | 30 | |
| `--lr` | 2e-4 | AdamW |
| Sampler | DDPM, full 1000-step reverse loop | |
| EMA | none | raw model weights used for sampling |

Command: `python video_diffusion.py train`

## Results
- **Model size:** 39.7M params (`checkpoints/ckpt_baseline_v0_e30.pt`, 151.6 MB)
- **Training time:** ~379 min (~6.3 h) for 30 epochs / 37,500 steps
- **VRAM:** ~4.1 GB / 8 GB at batch 16 (~50%, GPU at 100% util)
- **Final loss:** ~0.004–0.009 (oscillating), epoch-30 mean ≈ 0.00

### Loss convergence (mean MSE per epoch)
| Epoch | 1 | 2 | 3 | 5 | 10 | 15 | 20 | 25 | 28 | 29 | 30 |
|-------|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|-----|
| Loss | 0.03 | 0.01 | 0.01 | 0.01 | 0.01 | 0.01 | 0.01 | 0.01 | 0.01 | 0.00 | 0.00 |

**Key observation:** loss converges by epoch ~3 and flatlines for the remaining 27 epochs.
The epsilon-prediction MSE is at the noise floor for this config — additional epochs do
**not** improve sample quality. Quality is capacity/method limited, not undertrained.

## Qualitative baseline (visual reference)
- Per-epoch training previews: `samples_v0/sample_e{1..30}_{0..3}.gif`
- Same-seed comparison clips: `generations/v0_seed1234/`
- 2×4 v1-vs-v0 comparison grid: `generations/grid_v1_vs_v0_seed1234.gif`

Observed: digits are **recognizable** (e.g. 0, 1, 9) and bounce/reflect off the canvas
edges correctly, but strokes are **thin/wispy and slightly noisy** — the limit we aim to
improve. Compare `samples_v0/sample_e1_*.gif` (blurry noise) → `sample_e30_*.gif` (sharp-ish)
to see the training progression.

## What we change next (v1)
1. **EMA weights** — sample from an exponential moving average of the model (≈0.999 decay).
2. **Bigger model** — `--base 96` (≈2.25× params), still fits 8 GB.

Re-run with the same `--size/--frames/--digits/--length/--epochs` so the comparison is
apples-to-apples, then regenerate the comparison grid and diff against this baseline.
